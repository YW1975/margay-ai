# 为什么选 cos —— 对比飞书 / 钉钉 / 腾讯 WorkBuddy / openclaw / hermes

> 主流玩家都在向"会议→任务→督办→主动推送"收敛。诸葛喵的差异**不在功能清单,在定位与范式**。

---

## 一、赛道现状(诚实地看竞品)

| 产品 | 路线 | 已经很强的地方 |
|---|---|---|
| **飞书 智能伙伴 / aily** | AI 内嵌协同平台 | 会议全流程(声纹/实时纪要)、待办提取(负责人+截止)→ 飞书任务、督办系统、智能伙伴"主动交付"+定时任务、自定义智能体 |
| **钉钉 AI 助理 / 钉钉 One** | AI 内嵌协同平台 | AI 听记(任务/责任人/时间→任务卡片→待办)、四大智能体(消息/审批/日程/会议)秘书式主动排序、场景化助理矩阵 |
| **腾讯 ima + WorkBuddy** | 桌面执行型 Agent + 知识库 | ima=知识脑;WorkBuddy=本地执行手脚(感知→规划→执行→交付)、定时任务追踪、交付物推企微、"数字同事" |
| **openclaw / hermes-agent 类** | 通用自主 agent 框架 | 通用任务自动化、可编排、开发者向 |

**结论:把"会议纪要 + 待办提取 + 进度跟踪 + 主动推送"当卖点,已经是红海。** 诸葛喵不能在这条线上比拼功能数量,要在**更高一层**赢。

---

## 二、诸葛喵的六条差异化(护城河)

### 1. 定位:给**老板**的参谋长,不是给**全员**的助手
飞书/钉钉/腾讯都是**普惠平权**的员工生产力工具 —— 每个人都能用、做单点任务。诸葛喵是**专为老板/高管经营管理视角**设计:它默认从"什么最该做、谁掉链子、哪条业务线卡了"出发,而不是"帮我写封邮件"。
> **品类差异:executive chief-of-staff AI ≠ employee productivity AI。**

### 2. 主动性的**层级**不同
竞品的"主动" = 主动推送总结 / 定时任务 / 排序未读 —— 仍围绕**信息整理**。
诸葛喵的主动 = **替老板追结果**:自己算谁该催、带升级级别(提醒→求助→升级)催办、识别僵尸项/久议无果/谁过载。
> 别人主动**整理信息**,诸葛喵主动**盯人盯事追闭环**。

### 3. 治理优先(dual-loop / dry-run / 拍板)—— 别人没有的信任底座
给老板用、且会动到"对人催办、对外承诺"的 AI,**治理是敢被放进管理层的前提**。
- 诸葛喵所有对外动作**默认 dry-run(只起草不发)**,经 dual-loop 治理门(propose → critic → **老板拍板**);L4/owner-only/高风险**绝不自动发**。
- 竞品的 agent 要么在自己权限内自动做,要么是个人桌面执行,**没有"管理级治理 + 老板始终在控制位"这套**。
> 这是"放飞自动化"与"老板敢托付"的根本区别。

### 4. 执行图谱 + 关系智能(最难自建、最值钱)
诸葛喵有 **人↔事↔阻塞 的执行图谱 + 社交关系图谱**:谁过载、谁久未联系、谁反复承诺未兑现、瓶颈卡在哪、Top10 优先级。
竞品是 **task-list / 知识库** —— 没有这种**结构化的"组织执行 + 关系"全局视图**。这恰是老板最需要、最难自己维护的东西。

### 5. 跨宿主 + 厂商中立 —— 漂在平台之上,不被锁死
飞书 AI 只在飞书、钉钉只在钉钉、腾讯绑微信生态 —— 都把你**锁进自己生态**。
诸葛喵跑在 **margay / ccl / codex / claude code 多宿主**,用飞书当传输但不锁死,可同时接飞书+钉钉+微信。
> 它是**漂浮在各平台之上的"老板私人参谋长"**,而不是某家 IM 里的一个功能。

### 6. 数字员工形态(margay)
诸葛喵是一个**有人设、有判断、会自己干活的数字员工/数字参谋长**,不是嵌在套件里的 feature —— 你可以像配一名员工一样配它(人设 + 技能 + 定时),它自己按时巡检、出结果找你拍板。

## 三、一句话总结表

| 维度 | 飞书/钉钉/腾讯 AI | **诸葛喵 cos** |
|---|---|---|
| 服务对象 | 每个员工 | **老板/高管** |
| 本质 | 你手里的生产力工具 | **为你打工的参谋长** |
| 主动性 | 主动整理/推送信息 | **主动盯人盯事追闭环** |
| 治理 | 权限内自动 / 无管理级治理 | **dual-loop + 老板拍板** |
| 全局视图 | 任务列表 / 知识库 | **执行图谱 + 关系智能** |
| 平台 | 锁进自己生态 | **跨宿主、厂商中立** |
| 形态 | 套件里的功能 | **数字员工(有人设有判断)** |

---

## 四、margay + cos 的合力

- **margay** = 数字员工平台(跑 claude/codex/ccl 等引擎、装技能、定时任务、Employee 体系)。
- **cos(诸葛喵)** = 跑在其上的**主动式参谋长**(经营管理大脑 + 治理 + 图谱)。
- 合起来 = 一个**真正能帮老板省心省力做经营管理的、可治理的、跨平台的数字参谋长** —— 这正是**硅基智能导入企业的第一个落脚点**。
