# 听会 → 工作项 → 催办 → 图谱(端到端)

> 诸葛喵的旗舰场景:一场会从"开完"变成"办成"。喵吉进会听、自己抽工作项、写表、催办、画图谱 —— 全程结果回到飞书,老板只看飞书。

## 端到端闭环
```
远程飞书会议
  → 喵吉入会(lark-vc-agent +meeting-join,真入会记录 + 读会中事件)
  → 听会转写(BlackHole 抓 Mac 音频 → 阿里云 NLS 实时 ASR → 逐字稿)
  → 抽工作项(谁/做什么/截止/状态)→ 写飞书行动项表
  → 催办(cos-patrol:该催/找你定/只报告,dry-run 待拍板)
  → 执行图谱(人↔事↔阻塞,瓶颈/优先级)
  → 全部结果发回飞书群
```

## ① 听会
- **入会**:`lark-vc-agent +meeting-join <9位会议号>` —— 喵吉真进会议(飞书里可见),读"谁发言/谁加入"事件。
- **转写**:实时 ASR(阿里云 NLS)。喵吉听会走 **BlackHole 虚拟声卡**:你电脑输出设为「多输出设备」→ 声音同时进你扬声器 + BlackHole → 喵吉从 BlackHole 抓音频转写。
  - 设备:BlackHole 2ch(远程)/ 聚合设备=麦克风+BlackHole(现场)。

## ② 整理工作项
从逐字稿抽结构化行动项,写进飞书多维表格(Base)。字段:`事项 / 负责人 / 截止 / 状态 / 会议 / 会议日期 / 客户或项目 / 类型 / 可委派度 / 备注`。
> ⚠️ 写表必须 `lark-cli --as bot`(user 身份写会 91403 无权限);读可 user。

## ③ 催办
`cos-patrol` 读这张表 → 算 chase queue → 起草催办,分三类:**该催的 / 找你定的 / 只报告的**,每条带责任人。默认 dry-run,经治理门,老板拍板才发。

## ④ 图谱
把行动项 + 人 + 关系导出为图谱数据(`graph_viewer/export_graph_data.py`),web 可视化:人↔事↔阻塞、按业务线看状态、Top10 优先级、风险项。可截图/链接发回飞书群。

## 全飞书演示
整条链路的**用户可见输出全在飞书**:喵吉进会 → 发纪要 → 发工作项 → 发催办草稿 → 发图谱。老板不用碰任何 CLI/桌面 UI。
