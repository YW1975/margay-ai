# 数字员工

> 诸葛喵不是嵌在套件里的一个功能,而是一个**有人设、有判断、会自己干活的数字员工/数字参谋长**。你像配一名员工一样配它。

## 在 margay 里组装
margay 原生有 **Employee(数字员工)** 体系。把诸葛喵组装成一个 Employee:
- **人设/规则(persona)** —— 诸葛喵军师纪律:dry-run 默认、dual-loop 治理、三类输出(该催/找你定/只报告)、fail-closed。
- **启用技能(enabledSkills)** —— `cos-patrol` + `cos-gate` + `cos-setup`。
- **引擎(model)** —— Claude Code(margay 可选 Margay CLI / Claude Code / Gemini / Codex 作后端引擎)。
- **(可选)定时** —— 挂 margay 定时任务,每天自动巡检盯落实,不用你唤起。

## 创建路径
margay → 设置 → 数字员工 → ＋数字员工 → 填名称/描述/模型/规则 → 自定义技能勾 cos-*(每个因用 run_shell_command 需确认"启用")→ 确定。

## 已验证
创建后在对话里选「诸葛喵·军师」→ "巡检一下行动项" → 数字员工自己调 cos-patrol → cos_cli → 输出 `{queued, needs_approval, dry_run}` 并**应用人设**(打印"所有外发均未实际发出" + 分类解读)。

## 为什么"数字员工"是对的形态
- 老板要的不是"多一个工具",是"多一个**靠谱的人**替我盯着"。数字员工形态让诸葛喵像团队成员一样**被配置、被信任、自己干活、按时汇报、出事找你拍板**。
- 这也是"硅基智能导入企业"的具象:第一个进企业的硅基,是一个**会自己干经营管理活的数字参谋长**,不是一个聊天框。
